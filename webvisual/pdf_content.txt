1. Linux User Management
Understanding Users vs Root (user d/t root)
What does "d/t" mean? "d/t" is an abbreviation for "different
than" or "difference between". In this context, we're comparing
regular users with the root (superuser) account. 
Key Concept: User Accounts in Linux
Linux is a multi-user operating system, meaning multiple users can
access the system simultaneously. Each user has a unique identity
with specific permissions and restrictions. There are two main types
of users:

What is a UID? UID stands for "User Identifier" - it's a unique
numerical value assigned to each user account. UIDs below
1000 are typically reserved for system accounts (like services),
while regular user accounts start from 1000. The root user
always has UID 0, which is special and grants all privileges.

=== PAGE 4 ===
Listing Users
# List users
cat /etc/passwd
What is /etc/passwd? This is a system file that contains
information about all user accounts on the system. Each line
represents one user with details like username, UID, home
directory, and default shell. The file is readable by all users but
only writable by root. 
Switching Users
# Switch to another user
su - username
Breakdown:
su - Stands for "Substitute User" or "Switch User"
- - Dash flag that loads the user's environment variables
(login shell)
username - The user account to switch to
💡 
When you use su - username, you'll be prompted for that
user's password. If you use su - without a username, it
switches to root (requires root password).

=== PAGE 5 ===
Accessing Root / SuperUser DO (sudo)
What is "sudo"? "sudo" stands for "superuser do" - it's a
program that allows authorized users to execute commands as
another user, typically the root user. It's one of the most important
security features in Linux. 
Basic sudo Commands
sudo command          # Run single command as root
sudo -i               # Open root shell (interactive)
sudo su               # Switch to root user
whoami                # Check current user
id                    # Show UID, GID, groups
Breakdown:
sudo command - Executes a single command with root
privileges, then returns to your user
sudo -i - Opens an interactive root shell (you stay as
root until you type "exit")
sudo su - Switches to root user using sudo
whoami - Displays the current logged-in user
id - Shows detailed user information: UID, GID (Group
ID), and group memberships
How does sudo work? sudo works by checking a special file
called /etc/sudoers. This file contains a list of users and
groups who are allowed to use sudo, and what commands they
can run. Only users listed in this file can use sudo.

=== PAGE 6 ===
Managing sudo Access
sudo visudo           # Safely edit sudoers file
usermod -aG sudo username   # Add user to sudo group (Debian/Ubuntu)
usermod -aG wheel username  # Add user to wheel group (Arch/RHEL)
Breakdown:
sudo visudo - Opens the sudoers file in a safe editor
that checks for syntax errors
usermod - Modifies user account properties
-aG - Flags: "-a" means append, "-G" means add to
groups
sudo - The group name on Debian/Ubuntu systems that
grants sudo privileges
wheel - The group name on Arch/RHEL systems that
grants sudo privileges
✓ Always use sudo visudo to edit the sudoers file, not a
regular text editor. This prevents syntax errors that could lock you
out of sudo access!

Package managers automate the installation, updating, and
removal of software. 
apt — Debian/Ubuntu/Parrot/Kali
What is apt? "apt" stands for "Advanced Package Tool". It's the
default package manager for Debian-based Linux distributions. It
handles downloading, installing, updating, and removing software
packages from repositories (online servers that store packages). 
sudo apt update               # Refresh package list
sudo apt install <pkg>        # Install
sudo apt remove <pkg>         # Remove
sudo apt upgrade              # Upgrade all packages
Breakdown:
apt update - Downloads the latest list of available
packages from repositories (doesn't install anything)
apt install <pkg> - Downloads and installs a
package and its dependencies
apt remove <pkg> - Removes a package but keeps
configuration files
apt upgrade - Upgrades all installed packages to their
latest versions
💡 
Always run sudo apt update before installing new
packages to ensure you have the latest package information.

=== PAGE 8 ===
pkg — FreeBSD / Termux (Android)
What is pkg? "pkg" is the package manager for FreeBSD (a
Unix-like operating system) and Termux (a terminal emulator for
Android). It serves the same purpose as apt but for these
systems. 
pkg install <pkg>
pkg update && pkg upgrade
Breakdown:
pkg install <pkg> - Installs a package
pkg update - Updates the package list
&& - Logical AND operator (explained in Section 6)
pkg upgrade - Upgrades all packages
pacman — Arch Linux / Manjaro
What is pacman? "pacman" is the package manager for Arch
Linux and Manjaro. It's known for being simple, efficient, and
giving users more control over their system. The name is a
playful reference to the video game Pac-Man. 
sudo pacman -Syu              # Sync & upgrade
sudo pacman -S <pkg>          # Install
sudo pacman -R <pkg>          # Remove
sudo pacman -Ss keyword       # Search
yay -S <pkg>                  # AUR helper (like you use on your Arch setup)

What is AUR? AUR stands for "Arch User Repository". It's a
community-driven repository containing user-submitted package
build scripts. It allows users to easily build and install software
that isn't in the official repositories. 
3. Script Installation
What is a script? A script is a text file containing a series of
commands that the shell executes sequentially. Scripts automate
repetitive tasks and are commonly used for software installation
and system configuration. 
Making a Script Executable
# Make a script executable
chmod +x install.sh

# Run it
./install.sh
# Or run directly with bash
bash install.sh
# Run as root
sudo bash install.sh
Breakdown:
chmod +x install.sh - Changes file permissions to
make it executable
./install.sh - Runs the script from the current
directory
bash install.sh - Explicitly runs the script with bash
interpreter
sudo bash install.sh - Runs the script with root
privileges
What is chmod? "chmod" stands for "change mode". It's used to
modify file and directory permissions. The "+x" flag adds execute
permission to the file. 
Common Installation Script Pattern
#!/bin/bash
echo "Installing..."
sudo apt install zsh git curl -y
chsh -s $(which zsh)

Breakdown:
#!/bin/bash - Shebang line that tells the system to use

echo "Installing..." - Prints text to the terminal
sudo apt install zsh git curl -y - Installs
packages with automatic "yes" to prompts
chsh -s $(which zsh) - Changes the default shell to
zsh
What is $(which zsh)? This is command substitution. The 
which command finds the full path to the zsh executable, and
the $(...) syntax executes that command and uses its output
as an argument. So it finds where zsh is located and passes that
path to chsh. 
💡 
The -y flag automatically answers "yes" to all prompts,
useful for automated installations.

What is dpkg? "dpkg" stands for "Debian Package". It's the low-
level package manager for Debian-based systems. While apt is
user-friendly and handles dependencies, dpkg is more basic and
is actually used by apt behind the scenes. 
What is Snap and Flatpak? These are modern containerized
package formats. They bundle the application with its
dependencies in an isolated environment, making them work
across different Linux distributions. This solves the "dependency
hell" problem. 
5. Shell
What is a shell? A shell is a command-line interpreter that reads
your commands and executes them. It's the interface between
you and the operating system kernel. The shell processes your
input, finds the program you want to run, and executes it.

Shell-Related Commands
echo $SHELL         # See your current shell
cat /etc/shells     # List installed shells

Breakdown:

What is $SHELL? The dollar sign ($) indicates a variable in the
shell. SHELL is an environment variable that stores the path to
your current shell. Variables are containers that hold values you
can reference throughout your session.

6. Shell Features: OR, Piping,
AND (or, piping, and d/t)
What are these operators? These are special symbols that
control how commands are executed and how data flows
between them. They're fundamental to shell scripting and
command-line efficiency. 
AND operator && — Run second command only
if first succeeds
sudo apt update && sudo apt upgrade
How does && work? The AND operator executes the second
command only if the first command completes successfully
(returns exit code 0). If the first command fails, the second
command is skipped. This is useful for chaining dependent
commands. 
Example Explanation:
First: sudo apt update runs
If it succeeds: sudo apt upgrade runs
• 
• 

=== PAGE 16 ===
If it fails: sudo apt upgrade is skipped
OR operator || — Run second command only
if first fails
ping -c1 google.com || echo "No internet"
How does || work? The OR operator executes the second
command only if the first command fails (returns a non-zero exit
code). If the first command succeeds, the second command is
skipped. This is useful for fallback actions or error handling. 
Example Explanation:
First: ping -c1 google.com tries to reach Google
If it succeeds: echo "No internet" is skipped
If it fails: echo "No internet" displays the message
Piping | — Pass output of one command as
input to next
ls -la | grep ".sh"
cat /etc/passwd | grep root
ps aux | grep firefox
How does piping work? The pipe operator (|) takes the output
(stdout) from the command on the left and passes it as input
• 
• 
• 
• 

=== PAGE 17 ===
(stdin) to the command on the right. This allows you to chain
commands together to process data step by step. 
Example Breakdown:
ls -la | grep ".sh" - Lists all files, then filters to
show only those with ".sh" in the name
cat /etc/passwd | grep root - Displays the passwd
file, then filters to show only lines containing "root"
ps aux | grep firefox - Lists all running processes,
then filters to show only Firefox processes
What is grep? "grep" stands for "Global Regular Expression
Print". It's a powerful tool that searches for lines matching a
pattern and displays them. It's one of the most useful commands
for filtering output. 
Combining All Three Operators
sudo apt update && sudo apt install nmap || echo "Install failed"
How does this work? This complex command demonstrates
operator precedence: 
sudo apt update runs first
If it succeeds, sudo apt install nmap runs (due to &&)
If the install fails, echo "Install failed" runs (due to ||)
If the install succeeds, the echo command is skipped

Redirection (related d/t)
command > file.txt      # Write stdout to file
command >> file.txt     # Append stdout to file
command 2> error.log    # Redirect stderr
command &> all.log      # Redirect both
What is redirection? Redirection allows you to send command
output to files instead of the terminal. There are different types of
output: 
stdout (1) - Standard output (normal command output)
stderr (2) - Standard error (error messages)
Breakdown:
command > file.txt - Overwrites file with stdout
command >> file.txt - Appends stdout to file (doesn't
overwrite)
command 2> error.log - Redirects only error
messages to file
command &> all.log - Redirects both stdout and stderr
to file
💡 
The difference between > and >> is important: > overwrites
the file, while >> appends to it. Always double-check which one
you need!
