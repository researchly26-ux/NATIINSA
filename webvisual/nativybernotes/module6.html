Shell Features: OR, AND, Piping

Master the &&, ||, and | operators plus I/O redirection — the most powerful shell productivity tools.

&& AND || OR | PIPE > REDIRECT
01

AND Operator — &&

Run 2nd only if 1st SUCCEEDS

The && operator executes the second command only if the first command completes successfully (returns exit code 0). If the first command fails, the second command is skipped.

# Classic AND example: update THEN upgrade

sudo apt update && sudo apt upgrade

# Chain multiple commands

mkdir project && cd project && git init

# Compile steps (each depends on the previous)

./configure && make && sudo make install

How && works — step by step:

  1. sudo apt update runs first
  2. If it SUCCEEDS (exit code 0): sudo apt upgrade runs
  3. If it FAILS (non-zero exit code): sudo apt upgrade is SKIPPED

Exit codes: Every Linux command returns an exit code when it finishes. 0 = success, 1-255 = error. The && operator checks this code to decide whether to continue.

02

OR Operator — ||

Run 2nd only if 1st FAILS

The || operator executes the second command only if the first command FAILS (returns a non-zero exit code). Perfect for fallback actions and error handling.

# Test internet connection — show message if it fails

ping -c1 google.com || echo "No internet connection!"

# Try to create dir, or show error

mkdir /tmp/test || echo "Directory creation failed"

# Try command, fall back to alternative

vim file.txt || nano file.txt

How || works — step by step:

  1. ping -c1 google.com tries to reach Google
  2. If it SUCCEEDS: echo "No internet" is SKIPPED
  3. If it FAILS: echo "No internet" runs as the fallback
03

Pipe Operator — |

Pass output as input

The pipe (|) takes the output (stdout) from the left command and passes it as input (stdin) to the right command. This lets you chain commands together to process data step by step.

# List files, then filter only .sh files

ls -la | grep ".sh"

# Read passwd file, show only lines with "root"

cat /etc/passwd | grep root

# List all processes, find only Firefox

ps aux | grep firefox

# Count lines in a file

cat /etc/passwd | wc -l

# Chain multiple pipes

ps aux | grep firefox | awk '{print $2}'

What is grep?

grep stands for "Global Regular Expression Print". It searches for lines matching a pattern and displays them. One of the most powerful and commonly used Linux commands — especially useful after pipes for filtering output.

ls -la | grep ".sh"

Lists all files, filters to show only .sh files

cat /etc/passwd | grep root

Shows passwd file, filters to root lines only

04

Combining All Three Operators

sudo apt update && sudo apt install nmap || echo "Install failed"

This complex command explained:

  1. sudo apt update runs first
  2. If it succeeds: sudo apt install nmap runs (because of &&)
  3. If the install succeeds: echo "Install failed" is SKIPPED
  4. If the install fails: echo "Install failed" runs (because of ||)
05

I/O Redirection — > >> 2> &>

Redirection allows you to send command output to files instead of the terminal. There are two types of output: stdout (standard output, normal results) and stderr (standard error, error messages).

command > file.txt # Write stdout to file (OVERWRITES)

command >> file.txt # Append stdout to file (keeps existing)

command 2> error.log # Redirect stderr (errors) to file

command &> all.log # Redirect BOTH stdout and stderr

# Real examples

ls /nonexistent 2> errors.txt

echo "New log entry" >> app.log

apt update &> update.log

>

Overwrites the file with stdout. Careful — previous content is lost! Creates the file if it doesn't exist.

>>

Appends stdout to the file. Safer — keeps existing content and adds new output at the end. Perfect for log files.

2>

Redirects stderr only (error messages). The 2 is the file descriptor for stderr. Use to capture errors separately.

&>

Redirects both stdout AND stderr to the same file. The & means "both streams".

⚠️ Critical Difference: > vs >>

> OVERWRITES the file every time — all previous content is permanently deleted. >> APPENDS — safe for log files. Always double-check which one you need!
