Linux User Management

Understanding users, root, UIDs, creating and managing user accounts — the foundation of Linux security.

01

Understanding Users vs Root

Linux is a multi-user operating system — multiple users can access the system simultaneously. Each user has a unique identity with specific permissions and restrictions. d/t means "different than" — the comparison of regular users vs the root superuser.

Aspect Regular User Root User
Permissions Limited permissions Unrestricted access
Home Directory /home/username /root
System File Access Can't modify system files Can modify anything
UID (User ID) UID ≥ 1000 UID = 0

💡 Two main user types:

  • Regular Users — Limited scope, cannot install software system-wide or modify system configs
  • Root User — The superuser, UID = 0, god-level access to everything on the system
02

UID — User Identifier

UID stands for User Identifier — a unique numerical value assigned to each user account. Linux uses UIDs internally to track who owns files and processes.

0
Root
Always UID 0, grants all privileges
1–999
System Accounts
Reserved for services & daemons
1000+
Regular Users
Normal user accounts start here
Check your UID

# Show current user

whoami

student

# Show full user info: UID, GID, groups

id

uid=1000(student) gid=1000(student) groups=1000(student),27(sudo)

03

Creating a User

The useradd command creates new user accounts. You need root/sudo privileges to create users.

Creating a new user

# Create a user and their home directory

sudo useradd -m username

# Set the user's password

sudo passwd username

Command Breakdown — every part explained:

sudo

Execute with superuser privileges. Without this, the command would be denied.

useradd

The command to add a new user to the system. Creates entries in /etc/passwd, /etc/shadow, and /etc/group.

-m

Flag that creates the user's home directory automatically at /home/username. Without -m, no home directory is created on some distros.

username

The login name for the new user. Replace with the actual desired username (e.g., alice, bob).

passwd

Command to set or change the user's password. After running, you'll be prompted to enter the new password twice.

💡 Pro Tips for useradd:

# Add to a specific group during creation

sudo useradd -m -G sudo,developers alice

# Set a custom default shell

sudo useradd -m -s /bin/zsh alice

# Set an account expiration date

sudo useradd -m -e 2026-12-31 alice

04

Deleting a User

The userdel command removes user accounts. Crucial for maintaining system security — always remove accounts that are no longer needed.

Deleting a user

# Delete user and remove home directory

sudo userdel -r username

userdel

Command to delete a user account. Modifies /etc/passwd, /etc/shadow, /etc/group.

-r

Flag that removes the user's home directory (/home/username) and mail spool. Without -r, the account is removed but files remain.

⚠️ Important:

Files outside the home directory owned by a deleted user remain on the system, owned by the old UID number. Use find / -nouser to locate orphaned files after deletion.

05

Listing Users — /etc/passwd

/etc/passwd is a system file that contains information about all user accounts. Each line represents one user.

# View all users on the system

cat /etc/passwd

root:x:0:0:root:/root:/bin/bash

daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin

alice:x:1000:1000:Alice:/home/alice:/bin/bash

📋 /etc/passwd Format:

username : password : UID : GID : comment : home_dir : shell

The x in the password field means the actual password hash is stored in /etc/shadow (which is only readable by root).

06

Switching Users — su

su stands for "Substitute User" or "Switch User". It lets you log in as a different user from within your current session.

Switching users

# Switch to another user (requires their password)

su - username

# Switch to root (requires root password)

su -

su

Switch User — opens a shell session as another user. You'll be prompted for that user's password.

-

The dash loads the target user's full environment (environment variables, working directory set to their home). Without -, you keep your current environment.

07

sudo — SuperUser DO

sudo stands for "superuser do". It's a program that allows authorized users to execute commands as another user (typically root) without switching to that user. It's one of the most important security features in Linux.

Basic sudo commands

sudo command # Run single command as root

sudo -i # Open root shell (interactive)

sudo su # Switch to root user

whoami # Check current user

id # Show UID, GID, groups

sudo -l # List your sudo privileges

sudo -u alice command # Run as another user (not just root)

sudo command

Executes a single command with root privileges, then returns to your regular user. No need to know the root password — sudo uses YOUR password.

sudo -i

Opens an interactive root shell. You stay as root until you type exit. Simulates a full root login.

whoami

Displays the current logged-in user name. Quick way to verify who you're running as.

id

Shows detailed user information: UID, GID, and group memberships. More detailed than whoami.

How does sudo work?

sudo checks a special configuration file called /etc/sudoers. This file contains a list of users and groups allowed to use sudo, and what commands they can run. Only users listed in this file can use sudo.

sudo logs every command — great for auditing and security
After authentication, timestamps let you re-run sudo for ~15 min without password
08

Managing sudo Access

sudo visudo # Safely edit sudoers file

# Add user to sudo group (Debian/Ubuntu)

usermod -aG sudo username

# Add user to wheel group (Arch/RHEL)

usermod -aG wheel username

sudo visudo

Opens the sudoers file in a safe editor with syntax validation. If you make a syntax error, visudo will warn you instead of saving the broken file.

usermod

Modifies user account properties. Used to change groups, shell, home directory, and other attributes of an existing user.

-aG

Two flags combined: -a means append (don't remove from other groups), -G means add to supplementary groups. Critical: without -a, you'd remove the user from ALL other groups!

sudo / wheel

sudo is the group name on Debian/Ubuntu. wheel is the group name on Arch/RHEL. Both grant sudo privileges to members.

✅ Best Practice:

Always use sudo visudo to edit the sudoers file — NEVER a regular text editor like nano /etc/sudoers. If you introduce a syntax error without visudo's validation, you could permanently lock yourself out of sudo access!
