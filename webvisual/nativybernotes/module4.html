02

Compile from Source

Download source code and build the program yourself. Gives you maximum control — enable/disable features, optimize for your CPU. Requires build tools: gcc, make.

# Install build tools first

sudo apt install build-essential

# Configure, compile, install

./configure && make && sudo make install

./configure

Checks your system for required libraries and generates a Makefile customized for your hardware.

make

Compiles the source code into binary executables. Can take minutes on large projects.

sudo make install

Copies compiled binaries to system directories (/usr/local/bin) so they're accessible system-wide.

03

.deb Files — dpkg

dpkg stands for Debian Package — the low-level package manager that apt uses behind the scenes. .deb files are pre-built packages ready to install.

sudo dpkg -i package.deb # Install

sudo apt install -f # Fix missing deps after dpkg

sudo dpkg -r package-name # Remove

-i = install. Unlike apt, dpkg does NOT auto-resolve dependencies. If a dep is missing, run sudo apt install -f to fix broken deps.

04

.tar.gz Archives

A .tar.gz is a compressed archive. tar bundles files together, .gz compresses them with gzip — like .zip on Windows.

tar -xzf archive.tar.gz # Extract

tar -xzf archive.tar.gz -C /opt/ # Extract to specific dir

tar -czf myarchive.tar.gz folder/ # Create archive

05

Snap & Flatpak — Containerized

Both bundle the app with its dependencies in an isolated container — solving dependency hell. They work across different Linux distributions.

# SNAP

sudo snap install vlc | sudo snap remove vlc

# FLATPAK

flatpak install flathub org.videolan.VLC

flatpak run org.videolan.VLC

📦 Snap

  • • By Canonical (Ubuntu)
  • • Auto-updates in background
  • • Good Ubuntu integration

📦 Flatpak

  • • Strong sandboxing
  • • Flathub = largest app store
  • • Works on any distro
