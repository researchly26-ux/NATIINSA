01

What is a Script?

A script is a text file containing a series of commands that the shell executes sequentially. Scripts automate repetitive tasks and are commonly used for software installation and system configuration.

Scripts are used for:

  • → Software installation
  • → System configuration
  • → Automating backups
  • → Batch file operations
  • → Scheduled tasks (cron)

Script file types:

  • .sh — Shell script (most common)
  • .bash — Bash-specific script
  • .zsh — Zsh script
  • .py — Python script
  • No extension is also valid!
02

chmod — Making Scripts Executable

chmod stands for "change mode". It modifies file and directory permissions. Before you can run a script, you must make it executable with chmod +x.

chmod usage

# Make a script executable

chmod +x install.sh

# Check permissions after

ls -la install.sh

-rwxr-xr-x 1 user group 1234 Mar 21 install.sh

↑ The 'x' means execute permission is now granted

chmod

Change mode — modifies the permissions (read/write/execute) of files and directories.

+x

Adds execute permission to the file. The + means "add", x means "execute". Without this, Linux refuses to run the file as a program.

Understanding Linux Permissions:

03

Running Scripts — 4 Methods

# Method 1: Run executable from current dir

./install.sh

# Method 2: Run directly with bash interpreter

bash install.sh

# Method 3: Run as root with sudo

sudo bash install.sh

# Method 4: Run as sudo executable (after chmod +x)

sudo ./install.sh

./install.sh

./ means "from the current directory". Required because Linux doesn't search the current directory for executables by default (security feature). The . is the current dir, / is the path separator.

bash install.sh

Explicitly calls bash to execute the script. Doesn't require chmod +x. Useful when you can't change permissions.

sudo bash install.sh

Runs the script with root privileges. Used when the script needs to install software system-wide or modify system files.

04

The Shebang Line — #!/bin/bash

The shebang (also called hashbang) is the very first line of a script. It tells the operating system which interpreter to use to execute the script.

Shebang examples

#!/bin/bash # Use bash interpreter

#!/bin/sh # Use sh interpreter (more portable)

#!/usr/bin/env python3 # Use Python 3

#!/usr/bin/env zsh # Use zsh

#!

The shebang sequence. # is the comment character in bash, but when combined with ! at the very start of line 1, it becomes a special directive recognized by the kernel.

/bin/bash

The full path to the bash executable. The kernel reads this path and launches that program to execute your script.

05

Common Installation Script Pattern

A typical Linux installation script follows this structure. Every line explained:

install.sh

#!/bin/bash

echo "Installing..."

sudo apt install zsh git curl -y

chsh -s $(which zsh)

echo "Installing..."

echo prints text to the terminal. Used for progress messages so the user knows what the script is doing.

-y

Flag that automatically answers "yes" to all prompts. Essential for automated installations that run without user interaction.

chsh -s $(which zsh)

chsh = Change Shell. -s specifies the new shell path. $(which zsh) = command substitution (see below)!

06

Command Substitution — $(...)

Command substitution lets you use the output of one command as an argument to another. Syntax: $(command)

# which finds the full path to a command

which zsh

/usr/bin/zsh

# $(...) substitutes that output directly

chsh -s $(which zsh)

# Same as: chsh -s /usr/bin/zsh

How it works step by step:

  1. Shell sees $(which zsh) and executes which zsh first
  2. which zsh outputs /usr/bin/zsh
  3. Shell substitutes that output into the original command
  4. Result: chsh -s /usr/bin/zsh is what actually runs
