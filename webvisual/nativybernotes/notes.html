Core Linux Concepts

Understanding the Linux Ecosystem

Linux is technically a Kernel—the software layer that manages hardware resources. A complete operating system is referred to as a Distribution (or "distro"), which combines the kernel with GNU utilities and a user interface.

Kali Linux
The industry standard for penetration testing.
Parrot Security
A lightweight alternative focused on privacy.
Ubuntu
A versatile base for both servers and desktops.

The Universal Rule

"Everything in Linux is treated as a file."

This design philosophy means that whether you are interacting with a document, a hard drive (/dev/nvme0n1), a running process (/proc/[pid]), or a network socket, the interface remains consistent.

The Filesystem Hierarchy Standard (FHS)

The Linux directory structure is a tree starting at the Root (/). Understanding this layout is critical for both offensive and defensive operations.

Path Purpose Security Significance
/Root DirectoryThe base of the entire system.
/binUser BinariesContains essential tools like ls, grep, and bash.
/etcSystem ConfigsHigh Priority: Stores system-wide settings & hashes.
/homeUser DataTarget for data exfiltration.
/rootAdmin HomeThe ultimate goal (Superuser access).
/tmpTemp FilesRisk: World-writable; used to hide malicious scripts.
/var/logSystem LogsDefense: Essential for tracking intrusions.
/devDevice FilesInterface for hardware components.
/optAdd-on AppsLocation for third-party security tools.

Summary & Reflections

    FHS Mastery: Knowing where logs (/var/log) and configs (/etc) live is the first step in system defense.
